## ⚠️ Kritik (sakın unutma)

### 1. Selective read pattern engine'e disipline edilmeli

**Neden:** 30KB+ büyüyen `learned.md`, `decisions.md` ve uzun referans dosyaları her step başlangıcında ajana tam yüklenirse:

- Prompt cache invalidation artar (her küçük değişiklik ajanın gördüğü full payload'ı değiştirir)
- Token maliyeti 5x kabarır (30KB tam yükleme = ~7500 token; TOC + ilgili bölüm = ~1500 token)
- Latency artar

**Yapılacak:**

- Workflow .md'lerinde **"büyük dosyalar için önce TOC oku, sonra ihtiyacın olan bölümü selective oku"** talimatı her ilgili step'e eklenmeli.
- Engine'in ajan promptuna "şu dosyalar > 30KB, TOC ile başla" uyarısı koyma seçeneği — engine boot'ta her dosyanın boyutunu ölç, threshold'u aşanlar için ajan promptuna meta bilgi ekle.
- Selective read pattern:
  ```
  1. Read(file, offset=0, limit=30)         ← sadece TOC
  2. Grep("^## <Bölüm Başlığı>", file)      ← satır numarası bul
  3. Read(file, offset=N, limit=M)          ← sadece o bölümü oku
  ```
- TOC otomatik güncelleme: **Engine sorumluluğunda** — yeni ADR/learned eklediğinde TOC'a anchor satırı ekle (parse + insert). +operation-manager maintenance turuna bırakılmadı.

### 2. Prompt cache aktivasyonu — `--system-prompt` flag kontrolü

**Neden:** Aynı persona/workflow tanımı 50 task'ta kullanılırsa, ilk 1 kez tam fiyat + 49 kez cache hit (~%90 indirim) olabilir. Şu an kullanılıp kullanılmadığı belirsiz.

**Yapılacak:**

- `server/engine/executors/claude-cli-executor.ts`'i kontrol et:
  - Persona tanımı `--system-prompt` flag'iyle mi gönderiliyor, yoksa user prompt'a mı gömülü?
  - System prompt cache için Claude API'sinin `cache_control` parametresi kullanılıyor mu (cli-spawn üzerinden başka bir mekanizma olabilir)?
- AGY (Antigravity) executor için aynı kontrol — `antigravity-cli-executor.ts`.
- Codex executor için aynı.
- Aktif değilse aktive et. v3.1.0 release notu olarak vurgula ("prompt caching active → cost reduction up to 90% for repeat persona invocations").
- Test: aynı task'ı 3 kez koşturup token usage'i karşılaştır.
